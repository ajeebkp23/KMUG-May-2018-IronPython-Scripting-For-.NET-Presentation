IronPython

  • Implementation of the Python programming language for .NET Framework
  • built on top of the Dynamic Language Runtime (DLR)
  • Written in C#
  • Works with Winforms, WPF, .NET Core and Mono
  • Doesn't work C extension of CPython
  • However https://github.com/IronLanguages/ironclad

Scripting alternatives

  • PythonForDotnet(Pythonnet)
  • IronRuby
  • Roslyn scripting
  • ScriptCS

Why Scripting?

  • customers need dynamic styling or functionality based on their own rules
  • query the data store using the existing business logic
  • quickly test out design changes to an existing API
  • play around with a running application
  • didn't want to have build a user interface to interact with the API

Why Scripting?

  • developers wouldn't need to use Visual Studio to write code to test the API
  • may be useful with testing
  • low level information workers
  • to provide easy product extension to customers

Sample Code


/* bring up an IronPython runtime */
ScriptEngine engine = Python.CreateEngine();
ScriptScope scope = engine.CreateScope();

/* create a source tree from code */
var code = "print 'hello from python'"
ScriptSource source =
	engine.CreateScriptSourceFromString();

/* run the script in the IronPython runtime */
source.Execute(scope);
Console.ReadLine();
